# 东大报关单证工作台 · 正式版 v1.0 部署说明

## 一、系统功能
上传合同 → 多层验证AI识别 → 核对录入 → 一键生成全套报关清关单证(PDF) → 存档翻单。
支持出口票(中国→哈/乌)和进口采购票(哈国东大从中国采购)。电脑/平板/手机三端适配。

## 二、多层验证识别
- 层1 双通道：Claude"提取员"+"稽核员"两次独立识别(并行)
- 层2 交叉比对：金额/数量/编号/IBAN 逐字符比对
- 层3 规则引擎(纯代码)：金额连乘合计、IBAN mod-97、SWIFT格式、HS编码库匹配(吨袋6305.32/编织袋6305.33)、中外文合同号比对、日期逻辑、占位符扫描
- 层4 置信度融合：通过/核对/阻断三级，阻断字段必须修正才能出单证
- 省费用：环境变量 RECOG_CHANNELS=1 改单通道

## 三、单证语言(防混乱)
- 俄文单证(发票/装箱单/CMR/ДТ-ГТД资料表)：纯俄文，海关官方术语(ДТ按ЕАЭС各栏графы)
- 中文单证(报关单/申报要素/CO)：纯中文，按《填制规范》
- 发票/装箱单可切 俄/英/中；海关官方单证语言锁定不可改
- 银行名、规格、付款方式自动转译，俄·英文单证零中文混排

## 四、税率库(已核验 2026-06-13，可更新)
- 哈萨克斯坦：进口关税按 ЕТТ ЕАЭС；6305.32/6305.33 = 10%，6305.39 = 7%；НДС 16%(2026税法改革基准税率)
- 乌兹别克斯坦：НДС 12%；进口关税必须按10位码在 tarif.customs.uz 逐票核验，系统仅提供常用6305包装袋测算基准
- 对乌出口建议每票准备CO/原产地资料，并由брокер确认是否涉及额外关税、限制、认证或原产地审查
- 税率在"系统设置→税率库"可改；税费测算按每一行HS编码联动；超60天未核验概览页提醒
- 申报税率最终以海关、税务机关、брокер核定为准

## 五、系统设置(管理员)
公司信息 / API接口 / 税率库 / 业务配置(票号前缀、口岸库、贸易条款库、HS编码库、客户库、印章名) / 数据管理(备份/恢复/清空)。
可设管理密码：设置后进入"系统设置"页需输入密码。

## 六、Render 部署(5分钟)
1. 本文件夹上传到 GitHub 新仓库
2. Render → New → Web Service → 选该仓库
3. Build Command: 留空或 npm install ；Start Command: node server.js ；Instance: Free
4. Environment 环境变量：
   - ANTHROPIC_API_KEY = 你的Claude密钥(必填)
   - CLAUDE_MODEL = claude-sonnet-4-6 (可选)
   - RECOG_CHANNELS = 2 (可选)
5. 部署完成 → 打开网址 → 进"系统设置→API接口设置"填入该网址→测试连接
6. 手机"添加到主屏幕"当APP用

本版后端为零第三方依赖，只有 Node.js >=18 即可启动。本地测试可用：
HOST=127.0.0.1 PORT=3123 node server.js

## 七、本地安装 PWA
1. 本机启动：HOST=127.0.0.1 PORT=3123 node server.js
2. 用 Chrome / Edge 打开：http://127.0.0.1:3123
3. 地址栏出现"安装"图标时点击安装；或点页面底部"安装到本机"
4. 安装后可从系统应用列表/启动台打开。已缓存首页、app.js、manifest、图标；AI识别接口仍需联网和服务器 ANTHROPIC_API_KEY。

## 八、数据与多设备同步
- 本机使用 localStorage 保留近期操作，录入中的票据会自动保存草稿，刷新页面后可自动恢复。
- Render 配置 `DATABASE_URL` 后，客户、产品、Logo、设计稿、票据和生成记录可保存到 PostgreSQL。
- 多设备同步采用稳定资料 ID 和图稿合并规则；电脑与手机分别新增的图稿会合并，已删除图稿不会被旧数据恢复。
- 建议定期在"系统设置→数据管理"导出 JSON 备份。
